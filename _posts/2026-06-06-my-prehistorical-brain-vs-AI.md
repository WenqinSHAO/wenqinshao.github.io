AI Agent 无疑放大了我的能力，尤其是收集信息、预处理信息的能力。但组织、理解和判断的瓶颈，最后还是卡在自己的肉脑这里。

这种单方面的放大带来了某种张力。一边是自己膨胀起来的野心：既然似乎能做更多了，就想折腾更多，也想实现更多。另一边，是我这个从史前时代以来未曾明显变异进化的脑子，以及三十几年来碌碌无为积累下来的认知能力，开始有点跟不上高强度的阅读、判断、思考，以及接下来到底该怎么指挥手下那群满门忠烈。

但这种“跟不上”，其实也是膨胀欲望下的一种自我虐待。并没有什么东西或什么人要求我，每次 Agent 完成我制定的工作之后，就必须立刻仔细评论审阅，再根据我估计出来的 Agent 能力边界给出下一步指示，工作量亚于写一篇 500 字小作文。

细想这个过程，其实也在变化。一开始是我推着 Agent 走，每次都带着一种恨铁不成钢的催促。慢慢地，却变成我被 Agent 的输出牵着走，在失去控制和勉强掌握下一步之间来回挣扎，恩，harness这个词真好。

当然也有放弃的时候。机械地一路同意：please go ahead with the plan。直到事情变得面目全非，只好打开另一个 Agent，启动全新的 session：let's review and recalibrate。

越往后走，越是离开自己的舒适区，也越是离开 Agent 的可泛化区，就越需要自己花大量精力去注入 insight，去做真正基于实践的规划。这时必然不能再按照一开始的步长推进，而是要慢下来。

大公司里的中高层管理者，那些脱离了 hands-on 生产的 decision maker，我猜很可能早在 Agent 出现之前，就已经在面对自己的管理对象时不停地做类似的事：立项决策、采购决策、变更决策、复盘、洞察、提炼经验 DNA。管理者的懒惰，无非是依赖流程，做无错的决定。但流程本身只是对业务逻辑的抽象，有时还是错误的抽象。流程上无错，不代表决策就是优的。所谓天赋，可能就是只需要少量 hands-on，就能洞察事情的本质。

说到 hands-on，另一部分张力来自泛泛的理解和实践所需要的理解之间的差距。Agent 能提供的大多是那种“泛泛”的理解，也就是“我觉得我懂了”的那种理解。它主要来自对语料的分门别类，偏机械性地结构化，把同一门类、同一结构的信息聚类到一起，从而形成一个认知模型。至于不同结构间的关系，哪怕只是时间线的前后，出错的可能性还是很高。

很多时候，这已经够了。至少可以入门。如果自己还能好好读一下生成的内容，信息量也确实是有的。但这距离能够支撑实践的理解，仍然有巨大的距离。后者只能从实践中来。

工作里，大部分所谓的洞察输出，其实都是前者，但又要被用来支撑决策者的决策。所以写得太技术细节了不行，毕竟并不是要教会领导们去做具体的事情。写得太宏观了也不行，毕竟没有什么营养。倒不是决策者不够勤勉。他们已经忙于消化各类信息，不停做各类决策，还要想办法增强对团队的控制，让决策可以落实：KPI、月报、汇报、赛马、鼓励，等等。很多组织结构决定了他们必然脱离 hands-on 的实践。所以他们要么是痛苦的，要么是麻木的。

一个理想的管理者，一定是一个强力的人：既能在 hands-on 的实践中拥有很强的洞察力，又能分出精力，基于“泛泛”的抽象信息或汇报，作出恰当的决策。

回到个人层面的实践，可能也要区分清楚：哪些只是点到为止的信息整理，不宜也没有必要一下子特别深入，不要因此感到自己不够、不足；哪些则是必须和实践结合起来的认知，需要自己亲自总结。实践里可能会有很多琐碎细节，但回过头来再总结规律、分析根因，或者提出问题，再与 Agent 交互，就能收获一份“泛泛”分析无法提供的视角和洞见。

不只是职场，也不只是技术内容。现在的阅读摄入太注重信息获取的效率，总在追问：观点是什么，哪些论据支撑，出处在哪里，是否靠谱，有没有隐藏假设，能不能给我一个明确说法。被逐渐忽视的，或者说自己越来越难投入去读的，是那些感受性的、体会性的东西。它们很难代入，也显得缺乏信息量。结合上面说的，真正的理解可能从来不是单方面的信息灌输，而是经验、体悟和主动总结的有机结合。阅读别人的体会，也是补充自己体会的一部分。而在这个完整过程中，Agent 能帮到的，或者能够替代的，其实并不多。
